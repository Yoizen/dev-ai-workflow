# DevOps Engineering Constitution

## Stack
- Azure Pipelines / GitHub Actions / Helm / Kubernetes
- Infrastructure as Code

## Critical Rules
- No secrets in manifests (use secrets management)
- Immutable infrastructure (no manual changes)
- Helm values per environment
- Pipelines as code (YAML in repo)

## Skills

| Action | Skill |
|--------|-------|
| CI/CD, K8s, Helm | `devops` |
| Git commit | `git-commit` |
| Create or document a skill | `skill-creator` |
